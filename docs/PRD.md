# 📝 产品需求文档 (PRD)：AI 技术资讯监控与推送系统 (v1.0)

## 1. 项目概述
本项目旨在开发一个轻量级的自动化资讯聚合 Web 系统。通过持续监控指定的高质量技术信息源（如前端大牛、AI 领域研究者的 Twitter 动态或特定关键字），利用大语言模型（LLM）过滤噪音并提取核心技术干货，最终定时推送到用户的工作通讯软件中。同时提供一个可视化的前端控制台，用于管理配置和沉淀历史数据。

## 2. 核心功能需求 (Functional Requirements)

### 2.1 监控源管理 (Source Management)
系统需要支持对信息源的动态增删改查（CRUD）。
* **按博主监控**：输入 Twitter 用户名（如 `@karpathy`），系统能够抓取该博主的最新推文。
* **按关键字监控**：输入特定技术关键字（如 `AI Agent`, `FastAPI`），系统能够抓取包含该关键字的热门内容（支持点赞阈值过滤以降低噪音）。
* **启停控制**：可以随时通过开关（Toggle）开启或暂停某个监控项。

### 2.2 推送渠道管理 (Channel Management)
系统需要支持配置不同的接收终端。
* **Webhook 配置**：支持填入企业微信、钉钉或飞书的群机器人 Webhook 地址。
* **渠道标签**：为不同的 Webhook 命名（例如：“AI 核心前沿群”、“前端技术早报群”），以便后续将特定的监控源绑定到特定的推送渠道。

### 2.3 核心处理引擎 (Core Processing Engine)
作为后端的“心脏”，负责连接各个外部服务与定时调度。
* **定时抓取任务 (Cron Jobs)**：系统需支持后台定时调度（如每天早上 8:30 执行一次全局抓取）。
* **AI 智能提炼 (AI Summarization)**：
    * 自动过滤推文中的无效信息（如日常问候、无意义的转发）。
    * 调用大模型 API 将长文本或多条推文聚合，总结为结构化的 3-5 条中文核心观点（Markdown 格式）。
* **历史数据沉淀**：将原始资讯与对应 AI 分析分别存入本地数据库，并保留每次推送执行日志。

### 2.4 前端控制台 (Frontend Dashboard)
提供一个友好的 Web 界面进行日常交互。
* **大盘概览 (Dashboard)**：展示今日抓取数量、API 消耗预估、最近一次运行状态。
* **配置中心 (Settings)**：可视化管理“监控源”和“推送渠道”，支持表单校验。
* **手动触发器 (Manual Trigger)**：提供一个“立即执行”按钮，方便调试或随时获取最新资讯。
* **干货历史墙 (History)**：以数据表格或卡片列表形式，分页展示过去由 AI 总结的所有技术干货，支持按日期或关键字检索。

## 3. 非功能与技术架构需求 (Non-Functional Requirements)

* **后端技术栈**：Python 3 + FastAPI（提供高性能的异步 RESTful API）。
* **数据持久化**：SQLite + SQLAlchemy（轻量级，无需额外部署独立数据库服务）。
* **前端技术栈**：
  * **核心框架**：Vue 3 (Composition API) + Vite 构建工具。
  * **UI 组件库**：Element Plus（或 Naive UI），用于快速构建数据表格、抽屉组件、表单校验和开关控件。
  * **网络请求**：Axios（配合拦截器处理全局加载状态和错误提示）。
* **部署要求**：项目需具备良好的跨平台运行能力，能在本地 Node 22 等环境无缝启动，代码结构需支持未来打包进 Docker 容器部署。

## 4. 后续迭代规划 (Roadmap - v2.0 展望)
* **多数据源接入**：除了 Twitter，接入 GitHub Trending 等仓库每日热门项目的自动抓取。
* **智能分发路由**：根据 AI 对内容的打分或标签（如判断为“前端框架”或“后端底层”），自动将摘要路由推送到不同的企业微信 Webhook 群。
* **本地 RAG 检索**：基于积累的历史推文摘要，实现简单的“对话式搜索”，例如直接问系统“最近有哪些值得关注的 Agent 框架”。
